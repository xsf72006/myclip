# myclip

A lightweight, native clipboard history for macOS. Press **⌘⇧C** to pop up a
small panel showing your recent copies (text and images), search through them,
and pick one to paste. Single binary, no Electron, no background daemons other
than the app itself.

## Features

- **Global hotkey** to toggle the panel (default ⌘⇧C, configurable in Settings)
- **Auto-paste** — picking an item pastes it straight into the app you came from (requires Accessibility permission, prompted on first use)
- **Move-to-top** — picking an existing entry promotes it instead of duplicating
- **Search-on-open** — the search field is focused so you can type immediately
- **Text and images** — TIFF or PNG payloads are stored as PNG on disk
- **Skips passwords** generated by Apple Passwords, 1Password, Bitwarden, etc.
  by honouring the [nspasteboard.org](https://nspasteboard.org/) convention
- **Shows 10 most recent by default** — click **Show all** to expand to the full 50 (oldest dropped automatically)
- **Menu-bar only** — no Dock icon, no menu bar shown for the app itself
- **Auto-starts at login** — registers itself via `SMAppService.mainApp` (toggleable in Settings → Open at Login)
- **Cmd+C still works** — myclip uses a different chord and doesn't intercept system copy

## Requirements

- macOS **14 (Sonoma)** or later
- Building from source additionally needs Xcode **Command Line Tools**
  (`xcode-select --install`) — the prebuilt binary needs nothing.

## Install

### via Homebrew (recommended)

```sh
brew tap xsf72006/tap
brew install --cask myclip
```

### via download

1. Grab the latest `myclip-X.Y.Z.zip` from the
   [Releases page](https://github.com/xsf72006/myclip/releases/latest).
2. Unzip and drag `myclip.app` into `/Applications/`.
3. Open it once (Spotlight → "myclip" → Enter). The first launch may be blocked
   by Gatekeeper (the build is ad-hoc signed, not Apple-notarised) — right-click
   the app and choose **Open** to dismiss the warning.

Either way, the app puts a 📋 icon in your menu bar and registers itself as a
Login Item so it auto-starts on subsequent logins. Press **⌘⇧C** (configurable
in Settings) to pop up the panel.

### build from source

```sh
git clone https://github.com/xsf72006/myclip.git
cd myclip
make install     # builds, copies to /Applications, opens it
```

For development iteration:
```sh
make build       # produces dist/myclip.app
make run         # build and open from dist/ (won't register as Login Item)
make release-zip # build + ditto a .zip and print the SHA256
```

## Configure the hotkey

Click the 📋 menu-bar icon → **Settings…**

The **Recorder** field shows your current shortcut. Click it (it turns blue
and shows "Press shortcut · Esc to cancel"), then **physically press the combo
you want** — e.g. ⌘⌥K. myclip captures and applies it immediately. Esc cancels.

A shortcut must include at least one modifier (⌘, ⇧, ⌥, or ⌃); bare keys
can't be bound as global hotkeys.

The **Presets** row has ⌘⇧C, ⌘⇧V, ⌘⌥V, and ⌃⇧Space ready to one-click.

## Customising the filter

`Sources/myclip/Pasteboard+Filters.swift::shouldAccept` is the single decision
point for whether a clipboard event becomes a history entry. The shipped rules:

| Rule | Why |
| ---- | --- |
| Skip pasteboards tagged Concealed/Transient/AutoGenerated | password managers |
| Skip pure-whitespace text | avoids accidental triple-click pollution |
| Skip images > 5 MB | full-screen screenshots would bloat `~/Library` |

To add app-specific blocks or different size caps, edit the one function and
`make install` to roll it out.

## Where data lives

```
~/Library/Application Support/myclip/
    history.json        ← metadata (50 most recent items)
    img-*.png           ← image payloads
```

`Clear history` in the menu bar wipes both. Uninstalling does **not** delete
this folder so you don't lose history if you reinstall; remove it manually
if you want a clean slate.

## Uninstall

If installed via Homebrew:
```sh
brew uninstall --cask myclip
```

If installed manually or via `make install`:
```sh
make uninstall                     # in the cloned repo
# or just:
rm -rf /Applications/myclip.app
```

The macOS Login Items entry is cleaned up by the system within a few logins.
Your history at `~/Library/Application Support/myclip` is left in place; remove
it manually if you want a clean slate.

## Troubleshooting

**Auto-paste doesn't work.**
Auto-paste needs Accessibility permission. On first use macOS shows a system
prompt — open System Settings → Privacy & Security → Accessibility, switch
myclip on, then `make uninstall && make install` so the LaunchAgent picks
up the new trust state. Until you do this, picking an item just copies it
to the clipboard and you ⌘V manually.

**The hotkey doesn't fire.**
Another app may already own the same shortcut (some terminals grab ⌘⇧C).
Change it under Settings → presets, or pick a custom combo.

**My passwords show up in history.**
The skip rule relies on the source app tagging the pasteboard with one of
`org.nspasteboard.{Concealed,Transient,AutoGenerated}Type`. Apple Passwords,
1Password, and Bitwarden's desktop app all do; some browser extensions don't.
If a specific app misbehaves, you can add an app-specific block in
`Sources/myclip/Pasteboard+Filters.swift`.

**Logs.**
LaunchAgent stdout/stderr go to `/tmp/myclip.out.log` and `/tmp/myclip.err.log`.
